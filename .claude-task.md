## 需要完成的任务

### 1. 文档解析 API
创建 POST /api/upload/document 端点，支持上传 PDF/Word/TXT 文件。

功能要求：
- 接收 multipart/form-data，文件字段名为 "file"
- 可选的附加参数：session_id、action（summarize / analyze / qa）、question（qa 模式用）
- 解析文件内容：
  - PDF → 使用 PyMuPDF (fitz) 提取文本
  - Word (.docx) → 使用 python-docx 提取文本
  - TXT → 直接读取 UTF-8 文本
- 将提取的文本发给 Kimi API，要求 AI 进行总结/分析/回答
- 将 AI 返回的结果也保存到会话历史中（作为 assistant 消息）
- 返回 JSON：{ "success": true, "filename": "xxx.pdf", "analysis": "AI 的分析结果", "session_id": "..." }

### 2. 图片分析 API
创建 POST /api/upload/image 端点，支持上传图片文件。

功能要求：
- 接收 multipart/form-data，文件字段名为 "file"
- 可选的附加参数：session_id
- 支持的图片格式：.jpg, .jpeg, .png, .webp, .gif
- 使用 Kimi 的视觉模型分析图片（用 moonshot-v1-8k-vision-preview 或 moonshot-v1-32k-vision）
- 注意：Kimi 视觉 API 通过 content 数组传递，格式为：
  ```
  content: [
    { type: "text", text: "请描述这张图片" },
    { type: "image_url", image_url: { url: "data:image/jpeg;base64,..." } }
  ]
  ```
  需要将图片转为 base64 data URL 传入
- 返回 JSON：{ "success": true, "filename": "xxx.jpg", "analysis": "AI 的分析结果", "session_id": "..." }

### 3. PPT 生成 API
创建 POST /api/ppt/generate 端点（或 SSE 流式）。

功能要求：
- 接收 JSON：{ "topic": "...", "session_id": "..." }
- 调用 Kimi API 生成关于该主题的 Slidev Markdown 格式 PPT 大纲
- 可选流式返回（SSE 格式，类似 /api/chat/stream）
- 返回的 markdown 需要是 Slidev 格式（`---` 分隔每页）
- 已有 slidev_service.py 可将 markdown 转为 HTML
