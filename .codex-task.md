# Codex — 前端拓展功能 UI 任务

## 项目背景
本项目是一个「千禧梦」构成主义风格 AI 聊天网站。需要你在前端添加拓展加分功能的 UI。

**项目路径：** /root/weilaijishu-bighomework
**当前分支：** feature/extensions
**前端文件：**
- templates/index.html — HTML 模板（Jinja2）
- static/css/style.css — CSS 样式
- static/js/app.js — 全部前端逻辑（1562 行）

## 设计风格
本项目采用「构成主义」（Constructivism）设计风格，受 El Lissitzky 启发：
- 色板：革命红 #D62828 / 纯黑 #1A1A1A / 米白 #F5F0E6 / 靛蓝 #6366F1
- 粗重无衬线字体，斜置大字
- clip-path 对角裁切、对角线条纹背景、skew 变换
- 几何 SVG 图标（无 emoji，无外部库）
- 红色块/三角旋转动画

## 现有 UI 结构
templates/index.html 中的关键区域：
- 顶部导航栏 (topbar) — 带品牌标识
- 浮动侧边栏 (sidebar) — 会话列表
- 聊天区域 (chat-area) — 消息渲染
- 快捷指令按钮区 (quick-commands) — 当前有 @生成文档 和 @网页生成
- 输入区域 (input-zone) — input 输入框 + 发送/停止按钮
- 桌宠面板 (pet-panel) — 右侧浮动

## 需要完成的任务

### 1. 在输入区添加文件上传按钮
在 index.html 的 input-zone 区域，新增两个按钮：
- 📎 上传文档（PDF/Word/TXT）按钮
- 📷 上传图片按钮

样式必须匹配构成主义风格（红色/黑色/米白色，几何感）。

### 2. 文档上传功能（app.js）
实现 uploadDocument() 函数：
- 打开文件选择器（accept=".pdf,.docx,.txt"）
- 显示上传进度指示（一个小的加载动画）
- 调用后端 API：POST /api/upload/document (multipart/form-data)
- 将 AI 分析结果渲染到聊天区，就像一条机器人消息
- 添加一个 "📄 文档名" 的前缀指示

API 预期接口：
```
POST /api/upload/document
Content-Type: multipart/form-data
Fields: file=文件, session_id=currentSessionId, action=summarize
Response: { "success": true, "filename": "xxx.pdf", "analysis": "AI 的分析结果", "session_id": "..." }
```

### 3. 图片上传功能（app.js）
实现 uploadImage() 函数：
- 打开文件选择器（accept=".jpg,.jpeg,.png,.webp"）
- 显示图片缩略图预览
- 调用后端 API：POST /api/upload/image
- 将 AI 分析结果渲染到聊天区

API 预期接口：
```
POST /api/upload/image
Content-Type: multipart/form-data
Fields: file=图片文件, session_id=currentSessionId
Response: { "success": true, "filename": "xxx.jpg", "analysis": "AI 对图片的描述", "session_id": "..." }
```

### 4. PPT 生成功能（app.js 和 HTML）
- 在快捷指令区 (quick-commands) 增加一个 "@生成PPT" 按钮
- 或创建一个专用的 PPT 生成入口
- 输入主题后，调用 API：POST /api/ppt/generate
- 将生成的 PPT 大纲显示在聊天区，并提供 Slidev 构建按钮

API 预期接口：
```
POST /api/ppt/generate
JSON: { "topic": "PPT主题", "session_id": "..." }
Response (SSE): 流式返回 Slidev markdown 内容
或非流式: { "success": true, "markdown": "...", "session_id": "..." }
```

### 5. 消息区域扩展（可选的交互增强）
- 在已存在的复制按钮基础上，确认一键复制功能正常工作
- 为文档分析结果添加特殊样式（文件图标 + 文件名标签）
- 为图片分析结果添加缩略图显示

## 重要约束
- 不要使用任何外部 CDN 或 JS 库
- 所有图标必须用内联 SVG（参考现有代码中的 ROBOT_SVG、USER_SVG 格式）
- 新 UI 元素的样式必须与构成主义风格一致
- CSS 写在 static/css/style.css 底部
- JS 写在 static/js/app.js 底部
- 不要改动现有的核心聊天逻辑
- 使用原生 fetch API
